----- @ViewChild --------

    
@ViewChild is used in Angular to access a child component's properties and methods directly from a parent component.

It is useful when you want the parent to directly manipulate or read data from the child.
This is not recommended for all cases as it tightly couples the parent and child components, but it's useful when:
You want to call a child component's method.
You want to read or modify a child component's property.

Key Points
@ViewChild(ChildComponent) gives a reference to the child component instance.
We can call child.changeValue() or access child.value directly from the parent.
You should only access the @ViewChild in or after the ngAfterViewInit() lifecycle hook, because the child must be initialized.
